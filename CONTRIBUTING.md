## Code of Conduct

This project follows a basic Code of Conduct:
- Be respectful and inclusive
- No harassment, hate speech, or personal attacks
- Constructive feedback only

By participating, you agree to follow these rules.

## How Can I Contribute?

There are many ways to contribute:
- Reporting bugs
- Suggesting new features
- Improving documentation
- Writing or improving code
- Reviewing pull requests

---

### Reporting Bugs

Before reporting a bug:
- Check existing issues to avoid duplicates
- Make sure you are using the latest version

When opening a bug report, include:
- Clear title and description
- Steps to reproduce
- Expected vs actual behavior
- Screenshots or logs (if applicable)
- Your environment (OS, browser, version, etc.)

---

### Suggesting Features

Feature requests are welcome!
Please include:
- The problem you are trying to solve
- A clear description of the proposed feature
- Any alternatives you considered

---

### Improving Documentation

Documentation improvements are highly appreciated:
- Fix typos or grammar
- Improve clarity or structure
- Add examples or screenshots

You can submit documentation changes via a pull request.

---

### Contributing Code

1. Fork the repository
2. Create a new branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Make your changes
4. Commit your changes
5. Push to your fork
6. Open a Pull Request

## Commit Guidelines

Use clear and meaningful commit messages.

Recommended format:
```
type(scope): short description
```

Examples:
- `fix(auth): handle expired tokens`
- `feat(ui): add dark mode toggle`
- `docs(readme): update installation steps`

Common types:
- `feat` – new feature
- `fix` – bug fix
- `docs` – documentation changes
- `refactor` – code refactoring
- `test` – tests

Before submitting a PR:

- Follow the coding style
- Update documentation if needed

PR rules:

- Link related issues if applicable
